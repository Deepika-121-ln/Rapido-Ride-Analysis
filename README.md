# Rapido Ride Booking Data Analytics Using SQL and Power BI

## Project Overview

The objective of this project is to analyze Rapido's ride booking data and uncover key business insights related to bookings, revenue, customer behavior, ride cancellations, payment methods, and vehicle performance.
The project demonstrates the complete analytics workflow—from data preparation and SQL analysis to building an interactive Power BI dashboard for decision-making.


## Objectives

- Analyze ride booking trends.
- Measure overall business performance.
- Track booking completion and cancellation rates.
- Compare Bike and Auto ride performance.
- Understand customer payment preferences.
- Evaluate customer satisfaction using ratings.
- Generate insights to support business decisions.

# Dataset
The dataset contains ride booking information including:

- Booking ID
- Customer ID
- Vehicle Type
- Pickup & Drop Location
- Booking Status
- Ride Distance
- Ride Duration
- Booking Value
- Payment Method
- Customer Rating
- Driver Rating
- Cancellation Details


# Business Problems
### Booking Analysis
- ✔️ How many rides were completed successfully?
- ✔️ What is the average ride duration?
- ✔️ What is the average ride distance for each vehicle type?
- ✔️ How many rides were canceled by customers?

### Customer Analysis
- ✔️ Which vehicle type receives better customer ratings?
- ✔️ Who are the top 5 customers based on total bookings?

### Revenue Analysis
  ✔️ Which payment methods are most commonly used?
  ✔️ How much revenue was generated by Bike and Auto rides?

### Performance Analysis
   ✔️ What is the average ride time for each vehicle type?
   ✔️ What is the most common reason for incomplete rides?
